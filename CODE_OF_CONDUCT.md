# Code of Conduct

This project expects professional, respectful collaboration.

## Expected Behavior

- Be direct and technical without being hostile.
- Assume good intent, but stay precise about facts and risks.
- Keep reviews focused on code, documentation, tests, and user impact.
- Respect maintainer decisions about project scope and release readiness.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private contact information or credentials.
- Pressuring maintainers to disclose or ignore security issues.
- Repeated off-topic comments after maintainers ask to move on.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; reject
contributions; or block contributors who violate this policy. Security issues
should follow `SECURITY.md`, not public discussion threads.
